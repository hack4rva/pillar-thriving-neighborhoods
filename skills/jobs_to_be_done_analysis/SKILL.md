# jobs_to_be_done_analysis

Purpose: Identify the real user need behind a problem using JTBD framing.

## When To Use
- Problem feels vague or policy-heavy
- Multiple competing ideas exist
- Need clarity on user value

## Inputs
- Target user
- Context scenario

## Process
1. Write job statement: "When ___, I want to ___, so I can ___"
2. Identify current workaround behavior
3. Identify pain points in current solution
4. Define success criteria for user
5. Reframe solution around job completion

## Outputs
- Validated job statement
- Pain point list
- Solution direction tied to job

## Constraints
- Avoid feature-first thinking
- Focus on behavior, not demographics
