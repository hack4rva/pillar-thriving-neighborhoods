# civic_alignment_check

Purpose: Ensure solution aligns with Richmond MAP pillars and real city constraints.

## When To Use
- Before submission
- During judging prep
- When solution feels disconnected from real-world feasibility

## Inputs
- Solution concept
- Target MAP pillar

## Process
1. Map solution to pillar goal and success metric
2. Identify ownership:
   - Who in the City would own or champion this?
   - What existing system or workflow does it touch?
3. Define feasibility:
   - Near-term: can a pilot run within 90 days?
   - Long-term: what would scaled impact look like?
4. Identify missing data, permissions, or dependencies
5. Confirm solution ties to a real, measurable outcome

## Outputs
- MAP alignment statement
- Implementation pathway (pilot → scale)
- Risks and dependencies list

## Constraints
- Must tie to a real metric or outcome
- No purely theoretical solutions
- Avoid solutions requiring City system access or legislative action
