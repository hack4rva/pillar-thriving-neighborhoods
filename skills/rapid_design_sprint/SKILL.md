# rapid_design_sprint

Purpose: Run a compressed design sprint to move from problem → prototype within hours.

## When To Use
- Problem is ambiguous
- Team is stuck debating direction
- Need fast convergence before building

## Inputs
- Problem statement (preferably MAP-aligned)
- Target user (or guess if unknown)

## Process
1. Define the problem in one sentence
2. Map user journey (start → friction → outcome)
3. Individually generate solution sketches — no discussion
4. Vote on strongest concepts
5. Build lowest-fidelity prototype
6. Test with at least one human (mentor, peer)

## Outputs
- Selected solution concept
- Prototype (Figma, code, or mock)
- Key user feedback

## Constraints
- Timebox each phase aggressively
- No polishing before testing
