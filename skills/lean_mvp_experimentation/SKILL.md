# lean_mvp_experimentation

Purpose: Test a hypothesis using the smallest possible implementation.

## When To Use
- Team is overbuilding
- Need to demonstrate feasibility
- Competing solution directions exist

## Inputs
- Hypothesis
- Target metric or signal

## Process
1. Define hypothesis: "We believe ___ will result in ___"
2. Define MVP: landing page, mock API, manual workflow, etc.
3. Define success signal: click, signup, feedback, judge reaction
4. Build MVP (minimum effort)
5. Test and collect signal
6. Iterate once

## Outputs
- MVP artifact
- Validation result (signal observed or not)
- Decision (continue / pivot)

## Constraints
- No full systems
- No backend unless absolutely required
